# Plan: Redesign SettingsView.vue with BeerCSS Standards

## Context

The user wants to redesign the Settings page (`src/views/SettingsView.vue`) to follow BeerCSS (Material Design 3) component patterns. The project already uses BeerCSS v4.0.21 as a dependency, but the current settings page uses custom CSS instead of leveraging BeerCSS's built-in components.

## Current State

- **File**: `src/views/SettingsView.vue` (245 lines)
- Current settings sections: data storage path + update check
- Uses custom CSS classes (`.settings-section`, `.actions`, `.update-row`, etc.)
- Project has `QmButton` component wrapping BeerCSS buttons
- Theme mode cycling exists in title bar via `useTheme()` composable

## Approach

Rewrite `SettingsView.vue` to use BeerCSS's native component classes (cards, switches, buttons, grid/row layout) instead of custom CSS. This aligns the settings page with the rest of the app's BeerCSS usage.

### Changes to `src/views/SettingsView.vue`

1. **Template restructure**:
   - Wrap each settings group in BeerCSS `.card` components with `.surface-container` background
   - Use `.row` / `.grid` for layout instead of custom flex containers
   - Use BeerCSS `.button` classes directly (matching `QmButton` patterns: filled, tonal, border variants)
   - Add theme mode selector section using `useTheme().setMode()` — move from title bar to settings as a proper setting with radio/segmented control
   - Add BeerCSS `.field` containers for the path display

2. **Style cleanup**:
   - Remove most custom CSS — rely on BeerCSS utility classes
   - Keep only app-specific overrides (spacing, layout integration)
   - Use Material Design 3 surface elevation tokens from `tokens.css`

3. **Sections to include**:
   - **主题设置** (Theme): Light / Dark / Auto selector using BeerCSS segmented buttons or radio chips
   - **数据存储** (Data Storage): DB path display + change/reset buttons
   - **关于** (About): Version display + check for updates button

### Files to modify

| File | Change |
|------|--------|
| `src/views/SettingsView.vue` | Full rewrite of template + style sections |

### No new files needed

All BeerCSS components are CSS-only (no JS). The `useTheme` composable already has `setMode()` for direct mode selection.
